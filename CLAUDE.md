# Правила работы с проектом

## Архитектура

- Интерфейс и страницы — чистый HTML + CSS + JavaScript (без фреймворков, без CDN)
- Каждая страница — отдельный HTML-файл (`index.html`, `pricing.html` и т.д.)
- **Единственное исключение:** файл `api/analyze.js` — серверная функция (Vercel Serverless) для безопасной передачи запросов к Claude API
- **API-ключ Claude — только в переменных окружения Vercel**, никогда не в коде
- Деплой: Vercel (бесплатный тариф покрывает MVP)

## Дизайн

- **Фон:** тёмный (примерно `#1a1a1a` или близкий)
- **Текст:** светлый (`#f0f0f0` или белый)
- **Акцентный цвет:** оранжевый (`#ff6b35` или близкий тёплый оранжевый)
- Адаптивная вёрстка — всё должно корректно отображаться на мобильных устройствах
- Использовать `flexbox` или `grid`, `max-width` для контейнеров, `rem`/`%` для размеров

## Стиль текста в интерфейсе

- Простой и дружеский тон — как разговор с умным знакомым
- Никакого канцелярита: не «осуществить ввод данных», а «введите данные»
- Короткие предложения, конкретные действия
- Кнопки и подписи — живые и понятные без инструкции

## Процесс работы

- Перед созданием **любого нового файла** — сначала описать, что планируется сделать и зачем, и дождаться подтверждения
- Перед большими изменениями в `index.html` — кратко описать, что именно меняется
- Не добавлять функциональность, которая не была запрошена
- Не рефакторить код, который не трогается в рамках текущей задачи
