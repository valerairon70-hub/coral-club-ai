# Coral Club AI-ассистент — Описание проекта

## Что это за проект

AI-ассистент для дистрибьюторов компании Coral Club, который помогает расшифровывать диаграммы трекера здоровья (программа Бутаковой) и готовить персонализированные консультации для клиентов. Дистрибьютор загружает скриншот диаграммы и вводит 3 главных жалобы клиента — ассистент выдаёт готовый скрипт консультации с протоколом на продуктах Coral Club.

Проект строится с позиции нутрициолога и суплементолога: без медицинских диагнозов, но с экспертным объяснением причинно-следственных связей между состоянием систем организма и самочувствием клиента.

---

## Для кого

**Первичная аудитория (MVP):**
Валерий — дистрибьютор Coral Club, прошедший обучение по программе Бутаковой. Использует инструмент лично для подготовки к консультациям с клиентами.

**Вторичная аудитория (после MVP):**
Все дистрибьюторы Coral Club, прошедшие обучение по трекеру здоровья. Испытывают сложности с расшифровкой диаграмм и грамотной презентацией протоколов клиентам.

## Страницы продукта — что планируется

### 1. Главная — Анализ диаграммы
Основная рабочая страница.
- Загрузка скриншота диаграммы трекера (с телефона или компьютера)
- Ввод 3 главных жалоб клиента
- Кнопка «Получить анализ»
- Вывод результата: расшифровка + причинно-следственная цепочка + протокол + скрипт консультации

### 2. История консультаций
Журнал прошлых анализов.
- Список сохранённых консультаций с датой
- Возможность открыть любую консультацию заново

### 3. База знаний
Справочник для дистрибьютора.
- Описание 12 систем организма простым языком (норма / нагрузка / перегруз)
- Каталог ~200 продуктов Coral Club с механизмами действия
- Протоколы Бутаковой: какая система — какие продукты

## Технический стек

| Часть | Технология |
|-------|-----------|
| Фронтенд | Чистый HTML + CSS + JavaScript |
| Серверная функция | Vercel Serverless (`api/analyze.js`) |
| AI-модель | Claude API — claude-opus-4-6 (с поддержкой vision) |
| База знаний | Структурированные JSON-файлы (системы, продукты, протоколы) |
| Хранение истории | localStorage браузера (MVP) → база данных (следующий этап) |
| Деплой | Vercel (бесплатный тариф) |

**Почему Vercel Serverless, а не один HTML-файл:**
API-ключ Claude нельзя хранить в коде фронтенда — он будет виден любому в браузере. Серверная функция прячет ключ в переменные окружения и безопасно проксирует запросы.

## Обработка ошибок

| Ситуация | Что видит пользователь |
|----------|------------------------|
| Загружен не скриншот диаграммы | «Не могу прочитать диаграмму. Попробуйте другой скриншот» |
| Файл слишком большой (>5 МБ) | «Файл слишком большой. Максимум — 5 МБ» |
| Нет соединения с интернетом | «Проверьте подключение к интернету» |
| Ошибка на стороне Claude API | «Сервис временно недоступен. Попробуйте через минуту» |
| Превышен лимит анализов по тарифу | «Лимит исчерпан. Осталось X дней до обновления» |

**Принцип:** пользователь всегда видит понятное сообщение — никаких технических кодов ошибок в интерфейсе.

## Правовые рамки

- Ассистент не ставит диагнозы и не заменяет врача
- При перегрузе систем — мягко рекомендует обратиться к профильному специалисту
- Работает в рамках методологии и рекомендаций автора программы (Бутакова О.А.)
- Позиционирование: нутрициологическая поддержка, а не медицинская помощь
- В интерфейсе присутствует явный дисклеймер об этом
